Dependency

Dependency: string
  • Defined in contracts/Config.ts:33

A dependency is how we list service we depend on. These will be also be added in addition to anything listed in the tests.include block. This is a required array—think of this as the long-term way of stating dependencies. When you publish, relations will be inferred for/from any services listed here.

example

["api-service@1.0.0", "some-other-service@2.1.5"]

Tests

Tests: { contractFilePrefix?: string; include?: string[]; skipIncludingDependents?: boolean; skipTestingLocalResources?: boolean; skipTestingRemoteResources?: boolean }
  • Defined in contracts/Config.ts:38

Test setup.

Type declaration

  • Optional contractFilePrefix?: string

    An option to specify what the temporary generated contract file will be prefixed as. The contract itself will be generated as a TypeScript (.ts) file. You should remove any generated contracts in-between test runs to ensure you have no conflicts with old data.

    example

    "__quicktype-contract"

  • Optional include?: string[]

    An array of strings that specify what services and versions we want to test. Note that these are in addition to those listed in dependencies. This is an optional array—think of it as a temporary way of adding services you want to test, for example in CI. When you publish, the relations will not be inferred from services present here.

    example

    ["api-service@1.0.0", "some-other-service@2.1.5"]

  • Optional skipIncludingDependents?: boolean

    Should we skip including dependent services from our remote broker?

  • Optional skipTestingLocalResources?: boolean

    Should we skip testing tests found locally?

  • Optional skipTestingRemoteResources?: boolean

    Should we skip testing tests found on a remote resource?
